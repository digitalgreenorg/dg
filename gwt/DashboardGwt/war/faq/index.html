COCO FAQ

When you have any problems with COCO, we recommend that you check this FAQ for any specific suggestions we may have for your particular issue. If not, please follow the basic troubleshooting steps. If this fails to solve your problem, then contact Systems team.

COCO Setup

    How do I access COCO?
    COCO is available at http://www.digitalgreen.org/coco/home.html.
    Which software is required on the system for accessing COCO?
    1. Browser - Firefox 3.5. As of now, COCO is not supported in any other version of Firefox or in any other browsers. See installation instructions.
    2. Google Gears browser plugin. See installation instructions.
    How do I install software required for COCO Offline?
    1. Download and install Firefox 3.5 from here
    2. Download Gears file for firefox from here
    3. Open firefox and click on File menu which is on the top left corner of the firefox window.
    4. In the File menu, click on open file and load the gears file which you just downloaded. You will now see a window with a message saying "Install addons from authors whom you trust". Click on the "Install Now" button, which is present at the bottom of the window.
    5. The installation process asks you to restart Firefox. Go ahead and restart the browser.
    6. Do not update the browser under any circumstances. Optionally you can disable automatic browser updates. The instructions are present here.
    7. Now users can access COCO and use offline mode.
    How can I disable automatic browser updates in Mozilla Firefox?
    1. Open Mozilla Firefox.
    2. Go to "Tools" option in the menu bar.
    3. Click on the "Options" in the drop down menu. A pop-up will appear.
    4. Click on the "Advanced" panel.
    5. Click the Update tab. The auto-update options are displayed.
    6. To disable automatic checking for updates, deselect all three options -Firefox, Addons and Search Engine.
    I need a username and password for a new account. What should I do?
    Please send a request to system@digitalgreen.org via email. You will also need to send us the following details:
    • Name of the user requiring the account
    • Designation of that person
    • Organization to which he/she belongs
    • District in which he/she operates
    How can I add a desktop icon for COCO similar to Offline Gmail?
    • Reduce the size of your Firefox window so that you can see both the Firefox window and your Desktop
    • In Firefox, type www.digitalgreen.org/coco/home.html in the address bar.
    • Once the page loads, you will see a green colored DG icon next to the address in the address bar.
    • Drag this icon onto your Desktop (i.e. click on the DG icon, keep it clicked. Move your cursor over the Desktop. Once your cursor is over the Desktop release your mouse click button). You should see a shortcut to COCO with a DG icon on the Desktop.
    • When you click on this icon the COCO page with load in your default browser.
    • Make sure that Firefox 3.5 is your default browser.

    COCO Troubleshooting

      What are the basic troubleshooting steps?
      1. Please make sure your internet is working. Try going to www.google.com.
      2. Make sure that your system satisfies software requirements.
      3. If you're in 'Offline mode', go to 'Online mode'.
      4. Logout of COCO.
      5. Please clear browser cache.
      6. Please refresh-all (Ctrl + Shift + R) the COCO webpage.
      7. Email the Systems team. See how.
      How do I know which version of Firefox I am using?
      Click on "Help" tab. Click on "About Mozilla Firefox". The version number is given there e.g. 3.5.23.
      How to clear cache in browser?
      In Firefox, click on "Tools" tab, then click on "Clear Recent History". A window will pop up. Make sure you have selected "Everything" in Time range to clear and you have checked all the options in the small window.Then click "Clear now".
      How to clear Google Gears database?
      In Firefox,
      1. Go to "Tools" option in the menu bar.
      2. Click on the "Gears Settings" in the drop down menu. A pop-up will appear.
      3. Click on the "Remove" button next to the website "http://www.digitalgreen.org".
      4. Click on the "Apply" button.
      How do I contact Systems team?
      Please send an email to system@digitalgreen.org. In your email, please explain your problem and the which steps cause that problem in detail. Please try to ensure that these steps are specific enough that we are able to reproduce the problem on our machines, so that we can investigate and fix it. Please make sure you have gone through this FAQ, done the basic troubleshooting steps and any other steps specific to your error.
      Why can't I use COCO in latest browsers or Internet Explorer?
      COCO Offline mode requires Google Gears. Google has stopped supporting this technology and it is not available in IE or latest browser versions. A new version of COCO will be created that works without Google Gears.

      Going Offline

        How can I go offline?
        • Download the database by clicking on "Download" button. This process will take sometime. Please wait until the database is 100% downloaded. You might skip this step if you have downloaded the database previously. See this question.
        • Click on "Go Offline".
        Do I need to click on "Download button" every time I go offline?
        No, you don't need to click on "Download button" every time. If you have downloaded the database once, COCO will update this database whenever you enter new data. However, there might be cases when you need to download again. Following are the few cases:
        • There has been an update in the software used in COCO.
        • If some data has been modified on the server.
        • If your local database file is not complete i.e. you did not download the data completely.
        • Some data like Practices are shared across all users, you should periodically download a fresh database to get the latest data.

        COCO Download

          How to download a fresh database?
          • Clear your database. See how.
          • Login to COCO.
          • Click on Download.
          My download is stuck at 87%. What should I do?
          It's not actually stuck. Screening attendance is being downloaded at 87%. Usually, this data takes a long time to finish. Please wait patiently. If there is a problem with the download you would get an error message.
          Why am I getting an error message "This browser does not have the Gears plugin. Please install Gears and reload the application"?
          There are 2 possibilities:
          • You might be accessing COCO from a new version of browser. Please see the requirements of COCO.
          • Google Gears is not installed on your browser. See instructions. If you have already installed it, please uninstall and install again.
          • If you are sure that it is not one of the above, please email the Systems team.
          My download stopped in the middle. I got the error message "You may be experiencing server/bandwidth problems. Please try again, or contact support". What can I do?
          Please try the basic troubleshooting steps and try download again.
          I am getting an unknown error message. What do I do?
          Please try the basic troubleshooting steps and try download again.

          COCO data entry or data modification

            How can I add more than 25 farmers in Person groups?
            You can add up to 25 new farmers when you create a Person group from "Add Person group". If the group has more than 25 farmers, click on "Add person", enter the details of that person, and select the group that you just created.
            A farmer who was in one Person group has changed to a different one. How can I enter this data?
            1. On the COCO dashboard, click on "Persons".
            2. Find the person by entering their name in the search box.
            3. Click on the person to edit.
            4. Change the selection in Group list to the new group which the farmer has joined.
            5. Click Save.
            Some farmers in my region belong to more than one Person groups. How do I add this in COCO?
            COCO allows only one group per farmer. Ideally, you should select the most dominant group of the farmer. If you feel this is a much needed feature, we would encourage you to send a feature request with a strong case for why this is an important feature to system@digitalgreen.org.
            A farmer has recently joined a Person group, how can I add him or her?
            If the farmer already exists in the system, see this FAQ. Otherwise, do the following steps.
            1. On the COCO dashboard, click on "Add" next to "Persons".
            2. Change the selection in Group list to the new group which the farmer has joined.
            3. Click Save.
            A farmer left a group. How can I reflect this in COCO? Will previously entered screening and adoption data be affected?
            Find the farmer by clicking on "Persons" on Dashboard. Edit this farmer's data and select "None" under group section. The past screening and adoption data for this farmer will remain as it is.
            I selected a Person group in the Screening form, but Persons data is not loaded. What do I do?
            If you're operating in Offline mode, your database is incomplete. Please download the database again. If it still doesn't work, please try the basic troubleshooting steps. This should fix the problem. If it doesn't please email the Systems team.
            The data that I am seeing in COCO is not complete. Why?
            Either your internet connectivity is faulty or the Gears database on your computer is outdated. Please try the following.
            • If you're operating in Online mode, please try the troubleshooting steps.
            • If you're operating in Offline mode, your database is incomplete. Please download the database again by following this link.
            • If you tried above and are still facing the problem, contact systems team.
            I added some data in COCO, but it's not reflecting in Analytics. Why?
            If you are entering data in Offline more, please make sure you have uploaded it. Analytics does not count farmers who have not seen a screening. It also considers only those videos that are marked "Suitable for dissemination".
            How can I add or modify a Practice?
            The current list of practices is very comprehensive and should suffice most of the new video. Please make sure you have searched in the current list for the relevant practice. If you are sure, you need a new practice or edit an existing one, please send an email to systems with details.
            I saw a Practice earlier, it's not there anymore. How did this happen?
            Yes, this is possible. Agricultural practice is shared by all users. Sometimes based on request, we might modify or delete some practice.
            How can I delete some wrong data in COCO?
            Deletion is not allowed in COCO. There are 2 alternatives. First, you can always overwrite the data and fill in any new information, rather than using the 'Add new' page. Or, secondly, if there are many such entries, you can mail the system team to delete them for you.
            While editing or adding, why am I getting the error message "You don't have permission"?
            Based on the designation of user, certain data modification like Partner, State is restricted for some users. You'll have to contact your supervisor for adding data for which you don't have permission.
            I am trying to edit a Screening, but it's "Editing of Screening is not allowed in Offline mode". What's the solution?
            Editing of screening is only allowed in Online mode. Please click "Go Online" and edit this screening.
            I am trying to add a Screening in which a video from another region was shown, but COCO is not showing this video in the Videos screened list. How can I add this?
            You will see only those videos which are pertaining to your district. If the video is from another district, but from the same state, ask your supervisor to add this data. If the video is from another state, please send an email to Systems team with the name of the Video and Screening details.
            The "Add person page" does not allow me to add more than three adoptions ("Person Adopt Practices"). How can I add more than three adoptions for a person?
            You can only three adoptions using "Person" page due to system constraints. Please use "Person Adopt Practice" for adding extra adoptions.
            How can I assign more than three villages to a single animator?
            Please use the "Animator Assigned Village" page for assigning more than three villages to a single animator.
            Two Person groups attended a Screening. How can I add more than one Person group while entering the data for that Screening?
            Select the first Person group in the list. After the "Loading person" screen has disappeared. Now, keep the CTRL button pressed and select the second group. Please wait for the persons to be loaded for the selected groups.
            How do I add two farmers who have the same name and are in the same Person group?
            You should enter Father's name, while adding the details of the person. If you have entered this information, you will see father's name in brackets beside the Person name while entering Screening/Adoption data.

            COCO Upload

              How do I upload data?
              1. Please make sure that you are connected to Internet by typing www.google.com in the address bar.
              2. Click "Upload".
              While uploading data, COCO displayed this error message - "Validation Error. Please contact support". How can I complete my data upload?
              Validation errors are mainly caused by duplicate data entries. Please send the database file to System team for resolution. If you are on Windows using Firefox 3.5, as you should be, you will find your database file at C:\Users\{username}\AppData\Local\Mozilla\Firefox\Profiles\{profile}.default\Google Gears for Firefox. The {username} placeholder refers to the username used to sign into Windows. The {profile} placeholder is a alphanumeric string. If you need further information, please refer Google Gears Documentation.

              Data Inconsistencies

                What are data inconsistencies?
                In the past, we noticed some errors in the database due to human errors. We are trying to catch these errors and notify user about the errors using Data Inconsistencies. For example, sometimes the Video Production start data maybe after the Video Production end date. This is clearly an error in data entry. Another example is when a video takes more than 12 days to produce. Most of our videos are produced in well under that time and this may well be a data entry error. Therefore, this is flagged for the Field Officer's attention.
                How do I resolve them?
                • Review the message in the Error Msg column.
                • Click on the reference objects in the Object 1 and Object 2 columns and edit these appropriately.
                • Return to the Data Inconsistencies page and check the box in the "Not an Error" column.
                • Click "Save" at the bottom of the page.
                What is "Not an Error"?
                The Data Inconsistencies page is a tool for identifying and correcting errors. We identify errors based on simple logic and general trends. For example, most videos are produced in under twelve days and if a video takes more than twelve days to produce it is possible that this data was entered in error and the video actually took a shorter time to produce. However, some videos may as well take more than twelve days to produce. In this case, this data was not entered by mistake and is a valid and correct data entry. In such cases, we recommend that you mark these as "Not an Error" and click "Save".
                I resolved an error, but the number of data inconsistencies is still same. Why?
                Corrected data inconsistencies are removed every 24 hours. Please check after some time.
                How do I rectify the error if it says "This screening has video from other state, compared to the state of screening. This might be potentially wrong."? I don't understand what it means.
                Sometime, a user enters creates a screening which has video from other state. Sometimes, this is a valid case but most of times this is an error because videos are usually produced in local languages. Please click on the screening and review the Video selected.
